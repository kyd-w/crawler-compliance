# 爬虫数据采集合规规范

本技能提供中国网络爬虫数据采集的合规边界和操作清单。在开发任何爬虫/数据采集功能前，必须完成合规检查。

## 合规红线（触碰即刑事风险）

以下行为在任何情况下都不可实施：

1. **突破技术屏障入侵系统** — 非法侵入计算机信息系统罪（刑法第285条）
2. **绕过反爬措施获取数据** — 非法获取计算机信息系统数据罪
3. **高频请求导致目标服务瘫痪** — 破坏计算机信息系统罪（刑法第286条）
4. **大量爬取未授权的个人信息** — 侵犯公民个人信息罪
5. **爬取竞争对手商业秘密** — 侵犯商业秘密罪

## 三阶层合法性判定

数据采集必须逐层通过以下判定，任一层不通过则不可采集：

### 第一阶层：数据公开性

| 数据类型 | 可否爬取 |
|---|---|
| 公开数据（无需登录、无访问限制） | 可爬 |
| 半公开数据（需注册/登录但免费） | 受限，需遵守平台用户协议 |
| 非公开数据（付费、内部、需授权） | 不可爬 |

### 第二阶层：技术手段合法性

- 遵守目标网站 `robots.txt`
- 不绕过或破坏反爬措施（验证码、IP限制、频率限制）
- 不突破登录验证
- 不伪造身份（如实设置 User-Agent）
- 请求频率不影响目标服务器

### 第三阶层：使用目的合法性

- 目的必须合法、正当、必要
- 学术研究等公共利益目的可获更高宽容
- 商业化使用需额外评估
- 不得将数据用于与来源方竞争的产品

## 开发前强制合规清单

每个爬虫项目进入开发前，必须逐项确认。创建 TaskList 跟踪完成情况：

### 采集前评估

- [ ] 确认目标数据是否为**公开数据**
- [ ] 检查目标网站 `robots.txt` 是否允许爬取目标路径
- [ ] 检查目标网站用户协议/服务条款是否限制爬取
- [ ] 确认采集目的的合法性（用途描述记录留档）
- [ ] 确认数据是否包含个人信息 — 如包含则必须有合法授权基础

### 技术实现约束

- [ ] User-Agent 如实标识（包含项目名称和联系方式）
- [ ] 请求间隔 >= 3秒（或按 `robots.txt` 的 `Crawl-delay` 设置）
- [ ] 单IP请求频率上限：不超过 600次/小时
- [ ] 不使用任何反反爬技术（不绕验证码、不破解加密接口）
- [ ] 实现断路机制：连续错误时自动停止
- [ ] 设置总请求量上限，防止单次采集量过大

### 数据处理约束

- [ ] 个人信息字段存储前进行**脱敏处理**
- [ ] 数据仅用于声明的合法用途，不转售第三方
- [ ] 建立数据留存期限，到期自动清理
- [ ] 如涉及重要数据，需进行数据安全影响评估

### 运行维护

- [ ] 所有爬虫运行需记录日志（采集时间、数据量、目标URL）
- [ ] 定期检查目标网站 `robots.txt` 变更
- [ ] 收到目标网站停止请求通知后立即停止

## 数据源优先级

按合规风险从低到高选择数据源：

1. **政府公开数据**（最优先）— 各级政府数据开放平台、公共数据资源，通常有官方API
2. **行业公开数据** — 行业协会公开报告、企业官网公开信息
3. **第三方平台官方API** — 优先使用平台提供的API接口
4. **网页公开内容爬取** — 无API时严格按本规范执行

## 应急处理

| 情况 | 处理方式 |
|---|---|
| 收到目标网站停止请求通知 | 立即停止，评估影响 |
| 发现采集到非预期数据（含敏感信息） | 立即停止，删除已采集数据，评估影响 |
| 目标网站更新 robots.txt 禁止爬取 | 24小时内停止相关爬虫任务 |
| 爬虫异常导致目标服务受影响 | 立即停止，排查原因 |

## 执行流程

在开发爬虫时按以下顺序执行：

1. 收到爬虫/数据采集需求
2. **暂停编码**，先执行上述合规清单
3. 创建 TaskList 跟踪每项 checklist 的完成情况
4. 逐项确认并通过后，方可进入开发阶段
5. 代码实现中嵌入技术约束（频率限制、断路机制等）
6. 代码完成后验证所有约束是否生效
